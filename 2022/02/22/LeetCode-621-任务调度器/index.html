LeetCode - 621. 任务调度器

发表于 2022-02-22 分类于 刷题

题目

https://leetcode-cn.com/problems/task-scheduler/

解题思路

  1. 已知 tasks 元素的范围是26个大写字母，可以给定26个桶计算元素的个数。
  2. 然后按照个数倒序排列，因为仅需要得到时间，所以可以使用一维数组计算，但是此处使用 Comparator 进行倒序排序，不能传入基本数据类型，所以多了一列空列来占位。
  3. 排序后使用两个指针 i 和 j 指向实际的范围。
  4. 在实际遍历之前，需要知道，一个周期的长度是 n + 1 ，也就是 res % (n + 1) == 0 才是一个周期。
  5. 先从个数最多的元素 i 开始遍历，当遍历到 count[i][1] == 1 的时候，为了确保个数是从 j 到 i 逐步为0的，这个时候切换到 count[j] 开始计算，当 count[j] == 0，--j 。
  6. 此处有一个是否待命的判断：当 i >= j 也就是里面不为0的元素都遍历了一遍，一个周期还没有结束，这个时候 i 和 j 都不动，仅 ++res 直到这个周期结束。
  7. 执行下一个周期，则 i = 0。
