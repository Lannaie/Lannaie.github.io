LeetCode - 5. 最长回文子串

发表于 2021-12-25 更新于 2022-02-20 分类于 刷题

题目

https://leetcode-cn.com/problems/longest-palindromic-substring/

解题思路

1. 中心扩散法

这种方法应该是最容易想到的思路，唯一的问题是需要判断从当前位置为 left 出发，right 究竟是 i 还是 i + 1 长度更大。具体的思路如下：

  1. 单独提一个函数 centralExtension( String s, int left, int right ) 来计算从当前 left 和 right 出发的最长子串长度；
  2. 在主函数中遍历字符串的时候，比较以left = i 的时候，right = i 和 right = i + 1 的情况，哪个长度更大，取最大长度，然后与当前 maxlen = end - start 最大长度比较，如果比它大，就替换；
  3. 此处有一个小问题，为什么 start = i - (len - 1) / 2 ，因为我们直接获得了两种 right 情况的最大值，以最小的例子来看，left == right == 1 的时候，left = i - 1 / 2 = i 成立，left == i, j == i + 1 的时候，left = i - 2 / 2 = i - 1 不成立，可以推出，left = i - (1 - 1) / 2 == i - (2 - 1) / 2 == i 成立。

2. 动态规划

以 babad 为例，有如下步骤：

  1. 首先我们应该知道简单的状态转移方程：当 s.charAt(i) == s.charAt(j) 且 s.substring(i + 1, j) 即区间 [i+1, j-1] 的子串也是回文串，则当前区间 [i, j] 的字符串为回文串；

  2. 已知的边界条件：当 s.length == 1 的时候，肯定是回文串，如 a 、b 等，所以应该创建一个 dp 的二维数组，来存放当前字符串是否为回文串，初始化每个字符都是回文串：

  3. 并记录一个 maxlen = 1 和 start = 0 ；

  4. 接着逐步遍历长度为 2 ~ n 的子串，看是否有长度比 maxlen 更大的回文子串；

  5. 这里需要注意一点，当确定 s.charAt(i) == s.charAt(j) 时，有两种情况，第一种是 L <= 3 ，即 j - i < 3 ，这时当前 i, j 之间最多只有一个元素，而单个元素肯定是回文串，所以直接赋值 true ，如果 L > 3 ，则需要判断 [i + 1, j - 1] 这个区间内的子串是否是回文串，如果是，则当前 [i, j] 肯定是回文串。
