题目

41. 缺失的第一个正数

解题思路

​ 想要时间复杂度为O(n)，首先想到的就是哈希表存，然后遍历查找，但是这样空间复杂度就不是常数级别了，于是考虑使用数组来代替哈希表。

​ 先不考虑具体的元素值，单以数组的长度 len 来思考，这个最小的未出现的正整数出现的情况：只有两种，要么是在这个长度的范围内 i >= 1 && i <= len ，要么不在这个范围内 i > len 。

​ 接着不考虑具体元素的正负，如果数组中元素的值不是连续的1到len，那么就意味着 <= len && >= 1 的范围内有值没有被取到，这个值必然就是结果值；如果数组中的值排序后是连续的1到len，那么最小的正整数就必然是 len + 1 。

​ 基于上述思想，可以对原数组先进行一个排序，但是不应该是从小到大的执行升序排序，而是按照 nums[i] == i + 1 进行排序，也就是对数组 [3,-1,2] ，排序的结果应该为 [-1, 2, 3] ，这样我们遍历排序后的数组，当 num[i] != i + 1 就表示 i + 1 是当前最小的未出现的正整数。

hbase面试题

发表于 2022-03-03 更新于 2022-03-22 分类于 面试题

概念

​ HBase是一种分布式、可扩展、支持海量数据存储的NoSQL数据库，能够在HDFS上实现数据的增删改查。它在运行过程中需要不断切分和聚合，消耗的资源量让其在小数据集上不具有优势，但是对于大数据集能够做到亿万数据的秒级查询。并且Hbase是基于HDFS的，所以可以实现对hdfs存储数据做随机写操作。

​ HDFS是不支持随机写的，如果要对里面的数据进行修改，需要先下载下来，改完以后再删掉原来的数据重新提交，这种方式非常不灵活。
